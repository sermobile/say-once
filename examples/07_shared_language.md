# Пример 7: Аналитики как инопланетяне — общий язык

## Ситуация
Взаимодействие между командами с разной терминологией: аналитики, разработка, продукт.
Каждая группа живёт в своей модели мира и использует одни и те же слова по-разному.

## Как ломается коммуникация
Не потому что люди «плохо объясняют» — а потому что:
- разные модели мира
- одни и те же слова означают разное
- нет общего протокола взаимодействия

## Что помогает

**Минимум:** общая терминология на стыке зон ответственности.
Хотя бы на границе между командами слова должны означать одно и то же.

**Идеал:** формальный язык коммуникации — протокол, где:
- одни и те же слова означают одно и то же
- есть фиксированные статусы и этапы
- понятно, что считается результатом

## Пример: военная терминология NATO

```
Target identified → Engage → Shot → Splash → Target neutralized
```

Что даёт каждый этап:
- **Target identified** — все согласны, что именно является целью
- **Engage** — начато действие
- **Shot** — действие реально выполнено (не «собираемся», а уже сделали)
- **Splash** — ожидается результат (синхронизация)
- **Target neutralized** — зафиксирован итог

Ни одно слово не «примерное». Это не разговор — строго типизированный обмен состояниями.

## Аналогия
Коммуникация как API между системами: чёткие типы, чёткие статусы, чёткий контракт.
Когда протокол есть — количество недопониманий резко падает.

## Принцип
Коммуникация — это не просто обмен словами, это обмен состояниями.
Без общего протокола каждая сторона интерпретирует слова через свою модель мира.
