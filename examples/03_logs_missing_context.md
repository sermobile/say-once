# Пример 3: Логи без деталей ошибки

## Ситуация
В логах приложения появляется ошибка.

## Как дали информацию
```
ERROR: file not found
```

## Что было не так
Разработчику или дежурному приходится:
- гадать, какой файл искала программа
- лезть в код, чтобы найти место, где читается файл
- воспроизводить ситуацию, добавлять логирование, деплоить заново
- тратить время на расследование того, что могло быть очевидно сразу

## Как надо было
```
ERROR: file not found: /etc/app/config.yaml
```

Или ещё лучше — с контекстом:
```
ERROR: failed to load config: file not found: /etc/app/config.yaml
```

## Принцип
Сообщение об ошибке без конкретики — это не сообщение об ошибке, а сигнал тревоги без адреса. Всегда включай в сообщение **конкретные значения**, которые привели к ошибке.
